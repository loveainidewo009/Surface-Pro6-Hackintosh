## ⚠️ 开篇三条铁律（新用户必读，违反必翻车）

### 1. 序列号必须更换，否则 Apple 服务全废

本 EFI 已随附 OpenCore 三码（序列号、主板序列号、UUID）。**请务必使用 OpenCore Configurator 或 ProperTree 自行生成新序列号并替换。**

> **当前 EFI 中的序列号为占位符（FAKESERIAL / FAKEMLB / 00000000-0000-0000-0000-000000000000），必须替换为有效的真实三码，否则无法正常使用 Apple 服务。**

不换序列号的后果：

- iMessage、FaceTime 无法登录
- App Store 无法下载应用
- 与他人正在使用的设备产生冲突，可能被苹果列入黑名单（Blacklist）

### 2. 摄像头无解，请知悉

Surface Pro 6 的摄像头走 Intel IPU 总线，macOS 无原生驱动，相机功能无法使用。这是硬件层面的死结，请不要在这上面浪费时间。视频会议请使用手机或外接 USB 摄像头代替。

### 3. 内置 Wi-Fi / 蓝牙无解，必须外接

板载 Marvell 芯片在 macOS 下无驱动，且未来也不会有驱动。请直接参考下文 **【硬件补全方案】** 购买外设，不要再抱任何幻想。

## 三、硬件补全方案（解决 Wi-Fi / 蓝牙 / 接口不足问题）

**核心理念：** 既然内置无线网卡无解，不如通过外设补齐，顺便把 Surface 接口少的短板一次性解决。

### 方案一：全功能 Type-C 扩展坞（强烈推荐，适合固定场景）

这是我在家及办公室的主力方案。一个扩展坞同时解决网络、读卡、外接显示、充电四大需求。

- **推荐店铺：** 淘宝 CKTUTU 数码旗舰店
- **推荐型号：** 适用联想小新/拯救者的 Type-C【8合1】HDMI+网口+SD+TF+USB×2+C口+PD充电

**8合1 功能清单：**

| 接口 | 功能说明 |
|---|---|
| RJ45 千兆有线网口 | 即插即用，免驱，比 Wi-Fi 更稳更快，适合 NAS 传输和大型下载 |
| HDMI 接口 | 支持 4K@60Hz 外接显示器 |
| SD 卡槽 | 满足摄影、航拍玩家的读卡需求 |

> 💡 **为什么选这款：** 我实测过多个品牌的扩展坞，这款在 Surface Pro 6 上兼容性最好，PD 充电功率稳定，网卡不掉线。

## 四、触屏驱动（IPTSDaemon）详细安装教程

**为什么需要 IPTSDaemon？**

Surface Pro 6 的触摸屏在 macOS 下无法像普通 USB 触摸设备那样直接工作——它采用了一套名为 IPTS（Intel Precise Touch & Stylus）的私有协议，需要在用户空间运行一个守护进程来解析触摸数据并传递给系统。没有这个进程，触摸屏就是一块普通玻璃，点上去毫无反应。

### 📥 第一步：下载 IPTSDaemon

官方下载地址：<https://github.com/Xiashangning/IPTSDaemon>

1. 打开上述 GitHub 页面
2. 点击页面右侧的 **"Releases"** 进入发布页面
3. 下载最新版本的 `IPTSDaemon-xxx.zip` 文件（建议下载最新稳定版）
4. 解压 ZIP 文件，得到以下内容：
   - `IPTSDaemon`（主程序）
   - `install_daemon.sh`（安装脚本）
   - `uninstall_daemon.sh`（卸载脚本）
   - `iptsd.conf`（配置文件）

### 📂 第二步：运行安装脚本（推荐方式，最简单）

IPTSDaemon 官方提供了自动化安装脚本，这是最推荐的安装方式：

1. 打开**终端（Terminal）**

2. 进入解压后的 IPTSDaemon 文件夹目录：

```bash
cd /path/to/IPTSDaemon-folder
```

> 将 `/path/to/IPTSDaemon-folder` 替换为你实际的解压路径

3. 运行安装脚本：

```bash
sudo ./install_daemon.sh
```

正常情况下，脚本执行完毕后不会有任何输出，这表示安装成功。安装完成后，IPTSDaemon 会在开机时自动启动。

### 🔧 第三步：手动安装（如果脚本失败时使用）

```bash
# 1. 将 IPTSDaemon 主程序复制到 /usr/local/bin/
sudo cp IPTSDaemon /usr/local/bin/

# 2. 创建必要的目录
sudo mkdir -p /usr/local/lib

# 3. 将 plist 启动配置文件复制到 LaunchDaemons 目录
sudo cp com.xia.shangning.IPTSDaemon.plist /Library/LaunchDaemons/

# 4. 加载服务
sudo launchctl load /Library/LaunchDaemons/com.xia.shangning.IPTSDaemon.plist

# 5. 启动服务
sudo launchctl start com.xia.shangning.IPTSDaemon
```

### ⚙️ 第四步：自定义配置（可选）

所有可配置项均在 `iptsd.conf` 文件中：

1. 用文本编辑器打开 `iptsd.conf`
2. 根据需要修改配置参数（例如调整触摸灵敏度）
3. **重要：** 修改后必须删除每行配置前的 `#` 注释符，修改才会生效
4. 重新运行安装脚本以更新配置：

```bash
sudo ./install_daemon.sh
```

或者直接修改已安装的配置文件：`/usr/local/iptsd/iptsd.conf`

### 🔄 第五步：验证是否运行成功

```bash
# 查看进程是否存在
ps aux | grep IPTSDaemon
```

如果在进程列表中看到 `IPTSDaemon`，说明触摸驱动已成功运行。

### ✅ 验证触摸是否生效

重启系统后，用单指轻触屏幕：

- 如果鼠标光标跟随手指移动 → **成功！**
- 如果触摸无反应 → 请参考下方**【故障排查】**

### 🗑️ 卸载 IPTSDaemon（如需）

```bash
# 进入 IPTSDaemon 目录，运行卸载脚本
sudo ./uninstall_daemon.sh
```

### ⚠️ 故障排查

| 问题现象 | 解决方案 |
|---|---|
| 触摸无反应 | ① 确保使用最新版 BigSurface.kext 和 IPTSDaemon；② 手动运行 IPTSDaemon 查看错误输出；③ 检查 `sudo dmesg \| grep -E "SurfaceS\|SurfaceH\|SurfaceM\|IntelP"` 的输出 |
| 锁屏后触屏无法输入密码 | **Ventura 及以上：** 在「系统设置 → 锁定屏幕 → 辅助功能选项」中勾选「辅助功能键盘」；**Monterey：** 在「系统偏好设置 → 用户与群组 → 登录选项 → 辅助功能选项」中勾选「辅助功能键盘」 |
| CPU 负载偏高 | IPTSDaemon 处理触摸数据会占用约 5% CPU，属正常现象 |
| 休眠唤醒后触屏失灵 | 重启 IPTSDaemon：`sudo launchctl unload` 再 `sudo launchctl load` |
| 触摸漂移或乱跳 | 检查屏幕是否有污渍，用超细纤维布擦拭后重启 |

### 📌 重要提醒

- **必须配合 BigSurface.kext：** IPTSDaemon 需要与内核驱动 BigSurface.kext 协同工作，本 EFI 已包含该驱动
- **macOS 12 及以上需要 DebugEnhancer.kext：** 如需调试，需额外安装此驱动
- **CPU 负载说明：** 官方明确提示，处理触摸热图会增加约 5% 的 CPU 负载，属正常情况

## 五、R-Drive Image 7 全盘备份与恢复详细教程

**为什么需要 R-Drive Image 7？**

黑苹果的引导文件（EFI 分区）非常脆弱，Windows 更新、磁盘工具误操作、甚至一次不正常关机都可能破坏引导，导致双系统全部无法启动。用 R-Drive Image 7 做一次全盘备份，崩溃后 10 分钟即可恢复如初，比重新安装节省一个多小时。

R-Drive Image 7 是目前黑苹果社区公认最好用的备份工具，比 Paragon Hard Disk Manager 更轻量、速度更快，支持将整块硬盘（包括 macOS 的 APFS 分区和 Windows 的 NTFS 分区）打包为一个 `.rdr` 镜像文件。

### 📥 第一步：获取工具

R-Drive Image 7 提供 30 天全功能试用期，对于备份一两次完全足够用。如需长期使用，可自行购买正版授权。随 EFI 压缩包中也附带了该工具的安装包。

### 💾 第二步：创建全盘备份镜像

1. 启动 R-Drive Image 7
2. 在主界面点击 **"创建镜像"（Create Image）**
3. 选择源磁盘：选择 Surface Pro 6 的整块硬盘（通常是 Disk 0，包含 ESP 分区、Windows 分区、macOS 分区）
4. 选择目标位置：选择一个外接硬盘或大容量 U 盘（镜像文件大小约为实际已用空间的 60–70%，建议准备 128GB 以上的外接存储）
5. 命名并保存：给镜像起一个清晰的名称，例如 `Surface_Pro6_Full_Backup_20260618.rdr`
6. 点击 **"启动"（Start）**，等待备份完成（约 15–30 分钟，取决于数据量）
7. 备份完成后，将 `.rdr` 文件复制到至少两个不同位置（如移动硬盘 + 云盘），防止单点故障

> 💡 **进阶技巧：** R-Drive Image 7 支持在 Windows 下备份 macOS 分区，无需进入 macOS 即可操作，非常方便。

### ♻️ 第三步：从镜像恢复（系统崩溃时使用）

1. 启动 R-Drive Image 7（如果系统无法进入，可用 PE 启动盘运行 Portable 版本）
2. 点击 **"恢复映像"（Restore Image）**
3. 选择之前备份的 `.rdr` 镜像文件
4. 选择恢复目标：将镜像拖拽到目标磁盘（通常是原硬盘）
5. 点击 **"启动"（Start）**，等待恢复完成
6. 恢复完成后重启，双系统直接可用，无需重新配置 EFI

### ⚠️ 备份与恢复注意事项

| 场景 | 注意事项 |
|---|---|
| 备份目标盘容量 | 必须大于源盘已用空间，建议至少预留 128GB 空闲空间 |
| 恢复目标盘容量 | 目标磁盘大小不能小于源盘（无法将 256GB 备份恢复到 128GB 硬盘） |
| 恢复前确认 | 确保目标磁盘没有重要数据，恢复操作会覆盖整个磁盘 |
| 增量备份 | 可定期进行增量备份，节省空间和时间 |

## 六、潇然一键装机助理——Windows 备份与恢复详细教程

**为什么推荐潇然一键装机助理？**

与 R-Drive Image 7 搭配使用，两者各司其职：

- **R-Drive Image 7：** 负责**全盘备份**（macOS + Windows + EFI 一起打包）
- **潇然一键装机助理：** 负责**单独备份/重装 Windows 系统**（不碰 macOS 分区，保留双系统环境）

潇然一键装机助理（简称 XROK）是潇然工作室开发的一款 Windows 装机工具，支持在桌面环境或 PE 环境下在线/本地安装 Windows 系统，全面兼容 UEFI / BIOS、MBR / GPT 环境。

### 📥 第一步：获取工具

- **官方网址：** <https://sys.xrgzs.top/xrok>
- 随 EFI 压缩包中也已附带最新版安装程序

### 📦 第二步：仅备份 Windows 系统（推荐）

如果你只想备份 Windows 分区而不影响 macOS，请使用此功能：

1. 运行**潇然一键装机助理**
2. 在主界面选择 **"系统备份"**
3. 选择源分区：选择 Windows 所在的分区（通常是 C 盘）
4. 选择备份目标：选择外接硬盘或 D 盘（非系统盘）
5. 命名备份文件，点击开始
6. 备份完成后，文件格式为 `.wim` 或 `.esd`（高压缩比）

### 🖥️ 第三步：桌面环境下在线重装 Windows

**适用场景：** Windows 系统卡顿/中毒，但还能正常开机。

1. 关闭所有杀毒软件（特别是 Windows Defender 和 360，必须完全退出，火绒可以不用退）
2. 运行**潇然一键装机助理**
3. 选择 **"在线重装系统"**
4. 从列表中选择你想要的 Windows 版本（建议选最新版）
5. 勾选 **"安装一键急救系统"**，以便安装出错后可进入 PE 恢复
6. 点击开始，程序自动下载镜像并安装
7. 等待约 20–40 分钟完成

> ⚠️ **重要：** 请确保各盘符下的 `OSDownload` 文件夹为空，否则可能导致已下载的镜像被误删。

### 🔧 第四步：PE 环境下安装 Windows（最安全的方式）

**适用场景：** Windows 已完全崩溃无法开机，或需要彻底重装。

1. **制作 PE 启动盘：** 使用潇然 PE 或微 PE、优启通等无推广的 PE 工具制作启动 U 盘
2. **进入 PE：** 插入 U 盘，开机按住**音量减**键进入引导菜单，选择 U 盘启动
3. 运行**潇然一键装机助理（PE 版）**
4. 选择本地镜像（提前下载好的 `.iso` 或 `.esd` 文件）或选择在线重装
5. ⚠️ **特别注意：** 重装时只选择 Windows 分区（C 盘），**不要动 macOS 分区和 ESP（EFI）分区**，否则双系统会全部崩溃
6. 点击开始，等待安装完成

### 🚫 避雷警告

**请勿使用以下流氓 PE/装机工具：** 大白菜、老毛桃、电脑店、U启动、小白、韩博士等。

这些工具会：

- 在系统里植入流氓软件和广告
- 修改浏览器主页并锁定（木马级行为）
- 窃取系统信息参与后台刷量
- **破坏 OC 引导配置文件，导致 macOS 无法启动**

### 📋 潇然一键装机助理常见问题

| 问题 | 解决办法 |
|---|---|
| 提示"请退出杀毒软件" | Windows Defender 和 360 必须完全退出，火绒可以不用退 |
| 下载失败/速度慢 | 关闭 IPv6，使用公共 DNS（114.114.114.114 或 223.5.5.5） |
| Wi-Fi 密码丢失 | 工具会自动备份 Wi-Fi 密码到非系统盘的 `Xiaoran\WLANPassword` 目录 |
| 安装过程中弹出 CMD 窗口 | **切勿关闭！** 这是系统部署的必要进程，关闭会导致安装失败 |

## 七、安装与双系统维护指南（完整流程）

### 📥 准备工作清单

| 项目 | 说明 |
|---|---|
| 16GB+ U 盘 | 用于制作 macOS 安装盘 |
| macOS 镜像 | 推荐 Monterey 12.7（最稳定），也可尝试最新 26 版本 |
| BalenaEtcher | 将镜像写入 U 盘的工具 |
| 本 EFI 压缩包 | 包含 OC 1.0.7 引导配置和所有驱动 |
| 外接硬盘（128GB+） | 用于全盘备份（强烈建议） |

### 🔧 安装流程

#### 步骤 1：制作 macOS 安装 U 盘

1. 下载 macOS 镜像（`.dmg` 或 `.iso` 格式）
2. 使用 BalenaEtcher 将镜像写入 U 盘
3. 写入完成后，用 DiskGenius 或类似工具打开 U 盘的 ESP 分区

#### 步骤 2：替换 EFI 引导文件

1. 将 U 盘 ESP 分区中的原 EFI 文件夹删除
2. 将本教程附带的 **EFI 文件夹**复制到 U 盘 ESP 分区根目录
3. ⚠️ **重要：** 用 OpenCore Configurator 打开 `EFI/OC/config.plist`，在 `PlatformInfo` 下生成新的序列号、主板序列号、UUID。**当前 config.plist 中的三码为占位符（FAKESERIAL / FAKEMLB / 00000000-0000-0000-0000-000000000000），必须替换，否则无法使用 iMessage、FaceTime 和 App Store。**

#### 步骤 3：设置 UEFI 引导

1. 关机，插入 U 盘
2. 开机时按住**音量减**键（Surface 的 UEFI 引导菜单快捷键）
3. 选择 **USB 设备**启动

#### 步骤 4：安装 macOS

1. 进入 OC 引导界面，选择 **Install macOS**
2. 进入安装程序后，使用**磁盘工具**将目标分区格式化为 APFS 格式
3. 选择目标分区开始安装（约 30–45 分钟）
4. 安装完成后会自动重启，再次从 U 盘引导，选择新安装的 macOS 分区启动
5. 完成初始设置（语言、账户等）

#### 步骤 5：将 EFI 复制到硬盘

1. 进入 macOS 后，下载 OpenCore Configurator 或 MountEFI 工具
2. 挂载 U 盘的 ESP 分区和硬盘的 ESP 分区
3. 将 U 盘 EFI 文件夹**完整复制**到硬盘 ESP 分区
4. 拔掉 U 盘，重启测试是否能直接从硬盘引导

### 🪟 Windows 11 安装指南（双系统）

1. 下载 Windows 11 官方 ISO 镜像（24H2 或 26H2 均可，引导限制已破解）
2. 通过**启动转换助理（Boot Camp）** 或**磁盘工具**划分出一个 NTFS 分区用于 Windows
3. 使用**潇然一键装机助理**或 Rufus 将 Windows 安装到该分区
4. 安装完成后，OC 引导界面会自动识别 Windows 启动项
5. 首次进入 Windows 后，安装 Boot Camp 驱动（可从本 EFI 压缩包获取）

### 🛡️ 双系统备份策略（强烈推荐）

**组合方案：** R-Drive Image 7（全盘）+ 潇然一键装机助理（仅 Windows）

| 备份时机 | 工具 | 备份内容 | 备份位置 |
|---|---|---|---|
| 首次安装完成后（**必做**） | R-Drive Image 7 | 整个硬盘（含 ESP、macOS、Windows） | 外接硬盘 + 云盘 |
| 每月一次 | R-Drive Image 7 | 增量备份整个硬盘 | 外接硬盘 |
| Windows 大版本更新前 | 潇然一键装机助理 | 仅 Windows 分区（C 盘） | 外接硬盘或 D 盘 |

## 八、常见问题 FAQ

| 问题 | 解决办法 |
|---|---|
| 安装时卡在白苹果进度条不动 | 检查 U 盘是否插在左侧 Type-C 口；在 OC 界面按空格键选择 Reset NVRAM |
| 安装时提示"禁止符号" | 核显显存不足，已在 config 中预设 `framebuffer-stolenmem`，一般不会出现 |
| 进入系统后没有声音 | 检查声卡 `layout-id` 是否设置为 `28`（已在 config 中预设） |
| 触控板只能单指操作 | 重新安装 VoodooI2C 驱动，并确保 IPTSDaemon 正常运行 |
| Windows 无法引导 | 进入 OC 界面，选择 Reset NVRAM，重启后再试 |
| 休眠唤醒后黑屏 | 在 macOS 设置中禁用"显示器休眠"，或调整 `pmset` 配置 |
| 电池电量不显示 | 检查 ACPIBatteryManager.kext 是否加载（已在 config 中配置） |
| USB 网卡无法连接 | 检查是否已安装对应芯片的驱动（RTL8812BU 驱动在 EFI 压缩包中） |
| 升级 macOS 后部分功能失效 | 最新系统部分功能受限，建议回退至 Monterey 12.7 |
